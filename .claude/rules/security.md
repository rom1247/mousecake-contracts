# 安全规则

以下规则在编写或审查合约、脚本、测试时默认强制执行。

## 交互与敏感能力

- 与 ERC20/ERC721/ERC1155 交互时，考虑返回值、回调、恶意代币和非标准实现
- 对 `block.timestamp`、`block.number`、`msg.sender`、`delegatecall`、`selfdestruct` 的使用保持审慎
- 涉及提现、退款、奖励分发、回调交互时，优先考虑重入、防重复领取、累计记账误差
- 涉及签名验证时，明确域分隔、nonce、防重放和过期时间
- 涉及价格、汇率、预言机或外部状态时，明确精度、延迟、失效和信任假设
- 涉及升级、代理或初始化逻辑时，遵循 `upgradeability.md`

## 先停手再编码

发现以下情况时，不要直接编码，应先提醒用户风险：

- 需求可能导致资金冻结、权限失控或不可逆损失
- 需求会破坏既有存储布局或部署兼容性
- 需求依赖不可信外部合约但未定义安全假设
- 需求试图跳过测试、审计关键检查或必要访问控制
